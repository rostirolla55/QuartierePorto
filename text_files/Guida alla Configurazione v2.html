Guida alla Configurazione dell'Ambiente di Sviluppo

Questa guida illustra i passaggi necessari per impostare l'ambiente di sviluppo per la gestione dei contenuti multi-lingua del sito, utilizzando gli script Python per l'automazione.

1. Requisiti Fondamentali

Assicurarsi che il sistema sia dotato dei seguenti strumenti:

  1. Git: Strumento per il controllo versione (necessario per clonare il repository GitHub).

  2. Python 3.x: Linguaggio di programmazione (la versione 3.8 o superiore è raccomandata).

  3. Visual Studio Code (VS Code): Editor di codice moderno e versatile.

2. Configurazione di GitHub e Clonazione del Repository

Passo 2.1: Creazione dell'Account GitHub

  1. Creare un account su GitHub.

  2. NON è necessario inserire la password dell'account in Git ad ogni operazione. L'accesso sarà configurato al Passo 2.3.

Passo 2.2: Clonazione del Repository

Aprire il terminale (o Prompt dei Comandi/PowerShell su Windows, Terminale su Mac/Linux). Usare l'URL SSH del repository (che inizia con git@github.com:...).

git clone git@github.com:[il_tuo_utente]/[il_tuo_repo].git

cd [nome_della_cartella_del_repository]

Passo 2.3: Configurazione dell'Autenticazione Git/GitHub (Chiavi SSH)

Questo passaggio configura il "passaporto digitale" che permette al computer del collaboratore di comunicare con GitHub in modo sicuro e senza dover digitare continuamente la password.

A. Generazione delle Chiavi SSH

  1. Aprire il terminale (Git Bash su Windows) ed eseguire il seguente comando per generare una nuova coppia di chiavi. (Se richiesto, premere Invio per salvare nel percorso di default e, se non si desidera una password aggiuntiva, premere Invio due volte per un passphrase vuoto).

  2. ssh-keygen -t ed25519 -C "la_tua_email_github@example.com"

  3. Verranno generati due file:

    • id_ed25519 (la chiave privata e segreta).

    • id_ed25519.pub (la chiave pubblica da condividere con GitHub).

B. Copia della Chiave Pubblica

  1. Copiare il contenuto della chiave pubblica (id_ed25519.pub). Il modo più semplice è usare il comando del terminale (su Mac/Linux) o aprirlo con VS Code/Blocco Note.

    • Su Mac/Linux:

    • cat ~/.ssh/id_ed25519.pub

    • Su Windows (se si usa PowerShell):

    • Get-Content $HOME\.ssh\id_ed25519.pub

  2. Copiare l'intera riga di testo che inizia con ssh-ed25519 e finisce con l'indirizzo email.

C. Aggiunta a GitHub

  1. Accedere al proprio account GitHub.

  2. Andare su Settings (Impostazioni) -> SSH and GPG keys (Chiavi SSH e GPG).

  3. Fare clic su New SSH key (Nuova chiave SSH).

  4. Nel campo Title (Titolo), inserire un nome descrittivo (es. "Portatile di [Nome Collaboratore]").

  5. Incollare l'intera chiave copiata dal Passo B nel campo Key (Chiave).

  6. Fare clic su Add SSH key.

D. Test di Connessione

Eseguire questo comando nel terminale. Verrà chiesto di confermare la connessione (digitare yes):

ssh -T git@github.com

Se la risposta include la frase "Hi [Nome Utente]! You've successfully authenticated...", l'autenticazione SSH è configurata correttamente.

3. Configurazione dell'Ambiente Python e VS Code

L'automazione del progetto si basa su script Python (come add_page.py, extract_images.py e extract_gps.py).

Passo 3.1: Installazione di VS Code

Scaricare e installare Visual Studio Code.

Estensione Consigliata: Installare l'estensione "Python" di Microsoft dal pannello Estensioni di VS Code.

Passo 3.2: Installazione di Python

Se Python non è installato, scaricarlo dal sito ufficiale. Assicurarsi di selezionare l'opzione per aggiungere Python al PATH durante l'installazione su Windows.

Passo 3.3: Creazione e Attivazione dell'Ambiente Virtuale (Consigliato)

È buona pratica isolare le dipendenze del progetto. Eseguire i seguenti comandi nella cartella principale del progetto:

# Crea l'ambiente virtuale

python -m venv venv



# Attiva l'ambiente virtuale

# Su Windows:

.\venv\Scripts\activate

# Su Mac/Linux:

source venv/bin/activate

IMPORTANTE per VS Code: Dopo aver attivato l'ambiente virtuale (venv), aprire la cartella del progetto in VS Code. L'editor dovrebbe rilevare automaticamente l'ambiente virtuale e selezionare l'interprete Python corretto.

Passo 3.4: Installazione delle Dipendenze

I nostri strumenti di automazione (extract_images.py, extract_gps.py, convert_docx_to_html.py) richiedono librerie esterne specifiche.

Eseguire il seguente comando dopo aver attivato l'ambiente virtuale:

pip install python-docx Pillow piexif

Passo 3.5: Verifica e Scopo delle Librerie

  • python-docx: Gestione e conversione dei file Word in HTML.

  • Pillow: Elaborazione e lettura di base dei metadati delle immagini.

  • piexif: Estrazione dei metadati GPS (EXIF) per le coordinate geografiche.

5. Workflow: Aggiungere una Nuova Pagina

Passo 5.1: Preparazione

  1. Aprire la cartella del progetto in VS Code.

  2. Nella cartella data/translations/[lang], creare i file .txt puliti necessari per la nuova pagina.

Passo 5.2: Creazione della Pagina e Aggiornamento del Sito

Utilizzare il file Batch per eseguire l'automazione. Modificare add_paginaxy.bat con i parametri della nuova pagina (ad esempio, per la pagina "Museo"):

:: Modificare i valori qui sotto

SET PAGE_ID=museo

SET NAV_KEY_ID=navMuseo

SET PAGE_TITLE_IT="Il Museo Storico"

SET LAT=44.508000

SET LON=11.332000

SET DISTANCE=50

SET REPO_ROOT=..



python add_page.py %PAGE_ID% %NAV_KEY_ID% %PAGE_TITLE_IT% %LAT% %LON% %DISTANCE% %REPO_ROOT%

Eseguire lo script:

.\add_paginaxy.bat

Passo 5.3: Verifica e Commit

  1. Verificare che main.js, tutti i file HTML esistenti e tutti i texts.json siano stati aggiornati.

  2. Verificare che i 5 nuovi file HTML (museo.html, museo-it.html, -en.html, ecc.) siano stati creati.

  3. Una volta verificato che tutto sia corretto, eseguire il commit e il push delle modifiche su GitHub:

  4. git add .

  5. git commit -m "Aggiunta pagina: Museo"

  6. git push origin main

6. Configurazione di Google Analytics (GA4)

Per tracciare il traffico del sito, è necessario configurare una proprietà Google Analytics 4 (GA4) associata all'utenza Google.

Passo 6.1: Creazione della Proprietà GA4

  1. Accedere a Google Analytics con l'account Google desiderato.

  2. Creare una nuova Proprietà (selezionando GA4).

  3. Dopo aver creato la proprietà, selezionare "Flusso di dati" (Data Streams) e scegliere "Web".

  4. Ottenere l'ID di misurazione (Measurement ID) che inizia con G-XXXXXXXXXX. Annotare questo ID.

Passo 6.2: Inserimento del Codice di Tracciamento

Il codice di tracciamento GA4 deve essere inserito in tutti i file HTML principali.

  1. Aprire in VS Code tutti i file HTML base (es. index.html, index-it.html, index-en.html).

  2. Incollare lo snippet di codice di Google (gtag.js), sostituendo G-XXXXXXXXXX con l'ID di misurazione ottenuto al Passo 6.1, all'interno del tag <head> di ogni file, preferibilmente subito prima del tag di chiusura </head>.
